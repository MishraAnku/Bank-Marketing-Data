# A confusion matrix is a table used to evaluate the performance of a classification model by comparing actual values with predicted values.

# confusion_matrix() is a function from sklearn.metrics.

# It compares the actual values with the predicted values.
# It creates a table showing correct and incorrect predictions.

# Variables: A variable is a feature or value that can change and is used by the model to learn patterns from the data.

# y_test → Actual/True values from the test dataset

# y_pred → Values predicted by the model

# 📌 Result:
# The confusion matrix is stored in variable cm.

# sns → Seaborn library

# heatmap() → Draws a colored matrix

# cm → Confusion matrix data

# annot=True → Shows the numbers inside the boxes

# fmt='d' → Displays numbers as integers (whole numbers)

# CM It shows:

# How many predictions are correct

# How many predictions are wrong

# It compares:

# Actual values vs Predicted values

# 1️⃣ True Positive (TP)

# Model predicted Yes
# Actual value is Yes

# ✔ Correct prediction

# 2️⃣ True Negative (TN)

# Model predicted No
# Actual value is No

# ✔ Correct prediction

# 3️⃣ False Positive (FP)

# Model predicted Yes
# Actual value is No

# ❌ Wrong prediction

# Also called Type-1 Error

# 4️⃣ False Negative (FN)

# Model predicted No
# Actual value is Yes

# ❌ Wrong prediction

# Also called Type-2 Error

# 4️⃣ How Confusion Matrix is Measured

# It simply counts predictions.

# Does Confusion Matrix Use Training Data or Test Data?

# ✅ Usually it is calculated on the TEST DATA.

# Why Use Test Data?

# Because we want to check:

# 👉 How well the model performs on new unseen data

# what is the meaning of the Gini , Samples , Value in ML ?
# 1. Gini

# Gini means Gini Impurity. : It measures how mixed the classes are in a node.

# Formula idea: Probability of incorrect classification.

# Range: 0 → 0.5 (for binary classification)

# Meaning:

# Gini = 0 → Node is pure (all data belongs to one class).

# Higher Gini → Node has mixed classes.

# Example:
# If a node has only "Yes", then Gini = 0.

# 2. Samples

# Samples means the total number of data points present in that node.

# Example:

# Samples = 100

# This means 100 records from the dataset reached this node.

# 3. Value

# Value shows how many samples belong to each class at that node.

# Example:

# Value = [40, 60]

# Meaning:

# 40 → Class "No"

# 60 → Class "Yes"

# So the model will predict "Yes" because it has the majority.

# Memory Trick 

# | Term        | Meaning                                       |
# | ----------- | --------------------------------------------- |
# | **Gini**    | Measures impurity (how mixed the classes are) |
# | **Samples** | Total number of data points in that node      |
# | **Value**   | Number of samples in each class               |

# Decision Tree Pruning : Decision Tree Pruning means removing unnecessary branches (nodes) from a decision tree so that the model becomes simpler and avoids overfitting.
# Pruning cuts the extra branches of the tree that do not help in making better predictions.

# Steps of Decision Tree Pruning (Simple One-by-One)

# 1️⃣ Build the full decision tree

# Train the model using the training data.

# The tree grows until it fits the training data very closely.

# 2️⃣ Check the branches of the tree

# Look at branches that split the data but do not improve prediction much.

# 3️⃣ Evaluate branch performance

# Check whether removing the branch reduces accuracy or not.

# 4️⃣ Remove unnecessary branches

# If a branch does not improve the model, it is removed.

# 5️⃣ Replace it with a leaf node

# The removed branch is replaced with a single prediction node (leaf).

# 6️⃣ Test the pruned tree

# Evaluate the simplified tree on test data to see if performance improves.

# ✅ Final Result:

# Simpler model

# Less overfitting

# Better performance on new (unseen) data.
